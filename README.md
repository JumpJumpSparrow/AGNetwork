### 需求解析  

1.  request -> 充分定制网络请求，包括：  
  * 请求头
  * URL 定制灵活 
  *  参数传递 方便
  * 请求格式兼容 json 和 xm
  * 方便发起网络请求

2. response    
* 解析格式兼容 json 和 xml   
* 异常统计  
* 回调简单

### 设计思想  
将每次网络请求 看作一次 独立的封装，做到每次请求可定制。就像一串葡萄一样，所有葡萄都是完整的，都有共同的特征和茎，同时每粒葡萄都有自己的特征和个性。

### 
 这个框架是基于 AFN 的二次封装
